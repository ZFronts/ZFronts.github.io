2.1 编码格式：

2.1.1 python中常见的编码格式：

  • 内存编码：python解释器使用Unicode编码作为内存编码，简称内码；
  • 外存编码：在Python中，保存文件时可以使用不同的编码方式。默认情况下，Python使用UTF-8编码来保存文件；

2.1.2 python编码声明注释：

  • 作用：在文件开头加上编码声明注释，用以指定源码文件中的编码格式；
  • 语法：有以下三种形式
#encoding=字符编码名
#coding:字符编码名
#-*- coding:字符编码名 -*-

#例如规定.py文件使用ascii编码的代码如下：
#encoding=ascii
#coding:ascii
#-*- coding: ascii -*-

2.2 程序的格式框架：

2.2.1 单行单语句：

顾名思义，即一行一个语句；

2.2.2 单行多语句：

在同一行中使用多条语句，语句之间使用“分号;”分割，举例如下：

import math;print(math.pi);print(math.e)

2.2.3 多行语句：

  • 续行符：
    • 概述：Python提供“续行符”将单行代码分割为多行表达；
    • 表示：反斜杠</ins>
    • 注意：
      • 续行符后不能存在空格，续行符后必须直接换行；
      • 续行符不仅可以用于单行代码较长的情况，也适合对代码进行多行排版以增加可读性的情况；
(语句第一部分)\
(语句第二部分)\
……

#举例
n = 1 +\
    2 - \
    1 * 4\
    / 5
print(n)
  • 括号换行：
    • 小括号()、中括号[]、大括号{}；
#以“[]”为例
[(语句第一部分),
 (语句第二部分),
……,
(语句第n部分)]

2.2.4 缩进：

  • 指每行语句开始前的空白区域，用来表示Python程序间的包含和层次关系。Python采用严格的缩进格式以区分不同子代码块的执行；
  • 种类：（属于同一父代码块的子代码块需统一缩进种类）
    • Tab键缩进；
    • 空格缩进；
  • 缩进可能导致的问题：
    • 缩进错误“SyntaxError: unexpected indent”：由缩进种类不一致或缩进字符数量出错导致；
    • 逻辑错误：由子代码块对应错误的父代码块导致；

2.2.5 注释：

  • 定义：程序代码中解释器不解释运行的内容称为注释，一般用于程序员对代码的说明；
  • 分类：
    • 单行注释：以“#”开头，直到换行结束；
    • 多行注释：
      • 多个单行注释；
      • 纯三引号之间的内容；
#这是一个单行注释

'''这是一个
由三对单引号构成的
多行注释'''

"""这是一个
由三对双引号构成的
多行注释"""
  • 注意：多行注释不能嵌套使用，嵌套多行注释会导致语法错误。当你开始一个多行注释块时，Python 会一直将后续的行都当作注释，直到遇到另一组三个单引号或三个双引号。

2.3 命名约定及规范——标识符：

2.3.1 标识符的定义：

在Python语言中用于规范命名解释器能识别的可执行代码对象的名称；

2.3.2 标识符的分类：

  • 变量名：即变量的名字。变量是保存和表示数据值的一种语法元素，变量的值可以通过赋值运算符（“=”）进行修改；
a = 99
print(a)
a = 100
print(a)

# 99
# 100
  • 关键字（保留字）：指被编程语言内部定义并保留使用的标识符；
  • 函数名：即函数的名字；
  • 运算符：即运算符的符号；
  • 类名：即类的名字；

注：完整的标识符还包括格式控制符、#等；

2.3.3 命名约定：

  • 标识符只能由字母、数字、下划线、中文组成，且命名时数字不能被放在名称首位，虽然中文可以命名标识符，但是日常编程中更多还是用英文字母命名标识符；
  • Python语言大小写敏感，如a=0和A=0是两个变量；
  • 自定义标识符不能是Python保留关键字
  • 以下划线开头的标识符有特殊意义：
    • 单下划线开头（如_food）：代表不能直接访问的类属性，需通过类提供的接口进行访问；
    • 双下划线开头（如__food）：代表类的私有成员；
    • 双下划綫开头和结尾（如__food__）：代表Python中特殊方法专用的标识，如__init__()代表类的构造函数；

2.3.4 命名规范：

  • 解释器能识别；
  • 有利于程序员阅读；
  • 命名法：
    • 下划线命名法：
      • 采用英文小写单词，多单词组合时采用中间加下划线的方法来标识；
      • 例如apple_tree、user_name；
    • 驼峰命名法：
      • 小驼峰：
        • 要求除第一个单词之外，其他单词首字母大写
        • 例如appleTree、userName；
      • 大驼峰（帕斯卡命名法）：
        • 要求每个单词首字母大写，其他字母小写；
        • 例如AppleTree、UserName；
    • 匈牙利命名法：
      • 常用于C++类成员变量的命名（笔者）；
      • 基本原则：变量名=属性+类型+对象描述；
      • 举例（参考博客：四种常用的命名规则：帕斯卡命名法、驼峰命名法、下划线命名法、匈牙利命名法）

2.4 对象与变量：

2.4.1 对象：

  • 概述：Python中一切皆对象，所有对象都有一个布尔值；
  • 获取对象的布尔值——内置函数bool()：
    • 函数原型：bool(x)->bool
    • 解释：如果对象x为真，则返回True；否则返回False；（不传参也返回False）
  • 布尔值为False的对象：
    • False
    • 数值0
    • 空类型（None）
    • 空字符串（“”、str()）
    • 空列表（[]、list()）
    • 空元组（()，tuple()）
    • 空字典（{}、dict()）
    • 空集合（set()）
    • ……

2.4.2 变量：

  • 概述：变量相当于内存中一个带标签的盒子；
  • 定义语法：
变量名 = 值
  • 组成：
    • 标识：
      • 定义：为对象所存储的内存地址；
      • 获取——内置函数id()：
        • 函数原型：id(obj, /)->the obj's id
        • 解释：返回对象obj的内存地址
    • 类型：
      • 定义：类型表示的是对象的数据类型；
      • 获取——内置函数type()：
        • 函数原型：
          • type(object)->the object's type
          • type(name, bases, dict, **kwds)->a new type
        • 解释：
          • 返回对象object的数据类型；
          • 创建一个新类。name是一个字符串，代表新类的名称；base是一个元组，代表新类的基类，如果为空元组，则代表基类为object；dict是一个字典，包含类主体属性和方法的定义；
    • 值：标识对象所存储的具体数据，使用print()函数可以将值进行打印输出；

2.5 数据（字面值）：

2.5.1 定义：

在Python语言中除了标识符外，剩余的对象都可以叫做数据（字面值）。数据（字面值）是Python内置类型常量值的表示法；

2.5.2 程序代码组成：

  • 标识符（可执行命令）；
  • 数据（字面值）；

2.5.3 分类：

  • 字符串字面值；
  • 字节串字面值；
  • f-字符串（格式字符串字面值）；
  • 数值字面值；
  • 整数字面值；
  • 浮点数字面值；
  • 虚数字面值；

2.5.4 Python中的数制：

  • 十进制：Python中的纯数字默认表示十进制；
  • 二进制：
    • 拥有前导字符0b(B)的数字表示二进制；
    • eg.0b11表示二进制数11，十进制数3；
  • 八进制：
    • 拥有前导字符0o(O)的数字表示八进制；
    • eg.0o11表示八进制数11，十进制数9；
  • 十六进制：
    • 拥有前导字符0x(X)的数字表示十六进制；
    • eg.0x11表示十六进制数11，十进制数17；

2.6 表达式：

  • 概述：产生或计算新数据值的代码片段；
  • 作用：以表达单一功能为目的，运算后产生运算结果，运算结果的类型由操作符/运算符决定；
  • 组成：一般由数据和操作符等组成，是构成Python语句的重要组成部分；

2.7 Python命令行参数：

参考菜鸟教程——Python3 命令行参数

2.7.1 命令行参数：

使用命令行运行.py文件时，python后面的各字符（以空格分隔）均为命令行参数；

python test.py arg1 arg2 arg3
的命令行参数有test.py、arg1、arg2、arg3

2.7.2 利用sys模块获取命令行参数：

  • 获取命令行参数列表：sys.argv
  • 获取命令行参数个数：len(sys.argv)
  • 获取脚本名：sys.argv[0]

2.7.3 利用getopt模块来获取命令行参数：

  • getopt.getopt()函数：
    • 函数原型：getopt.getopt(args, shortopts, longopts=[])->opts, args
    • 参数说明：
      • args：是要解析的参数列表，不包括指向正在运行的程序的第一个元素。通常，这意味着使用 "sys.argv[1:]"；
      • shortopts：是脚本想要识别的选项单个字母组成的字符串，需要参数的选项后面跟着一个冒号:；
      • longopts：如果指定了longopts，它是一个包含应支持的长选项名称的字符串列表。选项名称中不应包含前导的--字符。需要参数的选项后面应跟一个等号=；
    • 返回值：
      • opts：是一个(opt, value)的列表。返回的每个(opt, value)对中，选项是第一个元素，前面有连字符（例如，-x，--name），选项参数是第二个元素，如果选项没有参数，则为空字符串；
      • args： 参数列表，包含那些没有 - 或 -- 的参数；
      • 注意：
        • 短选项参数如果需要参数，如果其后还有非空格字符，会默认将其后一个非空格字符至下一个空格之间的字符串作为它的参数；eg.假设-u需要参数，-h和--help不需要，那么-u --help -h和-u--help -h的-u的参数都是--help
        • 选项在列表中出现的顺序与它们被发现的顺序相同，因此允许出现多次；
        • 如果发现了非选项参数，其后的所有参数都将被视为非选项参数；
        • 需要参数的长选项参数在命令中可以输入“=”来指定参数值；eg.假设--user需要参数，那么--user=ming和--user ming的--user的参数都是ming
        • 长选项和短选项可以混合使用；
import getopt
import sys

print(getopt.getopt(sys.argv[1:],'n:u:',['name','user=']))

# 命令行为python main.py -u 1 -n 2 -u 3 --name --user 4 a b的输出为
# ([('-u', '1'), ('-n', '2'), ('-u', '3'), ('--name', ''), ('--user', '4')], ['a', 'b'])
